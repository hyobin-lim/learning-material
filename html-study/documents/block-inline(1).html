블록 레벨 & 인라인 레벨 태그 정리

블록과 인라인 사용 시 주의할 점

  • 인라인 요소는 줄 안에서 배치되므로, width/height 및 상하 margin을 직접 지정하는데 제약이 있을 수 있습니다. — 크기 지정이 필요하면 display:inline-block 또는 display:block 사용합니다.
  • 블록 레벨 요소는 다른 블록 요소와 인라인 요소를 포함할 수 있습니다.
  • <a> 태그를 예외적으로 제외한 나머지 인라인 요소는 블록 요소를 포함할 수 없습니다. 인라인 내부에는 주로 문장(phrasing) 요소만 넣습니다.
  • br은 인라인 요소 이므로 <br><ul>과 같이 바로 뒤 자식요소로 블록 요소가 오면 안됨
  • <li><br>은 가능 하지만 li는 블록 요소로 줄이 기본적으로 바뀌므로 br을 넣는 의미가 없음
  • 이미지(<img>)에는 항상 alt 속성을 넣어 접근성 확보.
  • 블록 요소는 기본적으로 줄바꿈이 발생하므로, 불필요하게 중첩하거나 남용하지 않도록 주의합니다.
  • 클릭 가능한 요소는 가능한 <a>나 <button> 사용. <div>에 onclick만 쓰면 키보드 접근성/역할 정보 부족 — role/tabindex 추가 필요.
  • <a>나 <button> 은 둘다 클릭을 요하는 블록 요소 이므로 같이 사용하면 안된다.
  • 인라인 요소는 레이아웃을 나누는 용도로 사용하지 말고, 텍스트나 콘텐츠 일부를 꾸미는 용도로만 사용해야 합니다.
  • 레이아웃 목적에는 시맨틱 태그(<section>, <article>, <header>, <footer>, <nav> 등)를 우선 사용하여 적절히 활용하면 구조와 의미를 명확히 할 수 있습니다. div는 보조 용도로 사용합니다.
  • CSS로 블록/인라인 성격을 변경할 수 있지만, 본래의 의미와 용도를 고려하여 사용하는 것이 바람직합니다.

마크업 규칙

쉬운 이해

  • 블록 태그 : 넓은 상자처럼 공간을 독점. 다음 요소는 아래에 배치됨.
  • 인라인 태그 :문장이나 텍스트 흐름 안에서 특정 부분을 꾸미거나 기능을 추가할 때 사용합니다. 옆에 다른 요소 배치 가능.
  • 즉, 큰 틀은 블록 요소로 잡고, 세부적인 꾸밈이나 기능은 인라인 요소로 처리하는 것이 일반적인 패턴입니다.

하위 블록 포함 가능 태그

HTML 블록 레벨 요소와 인라인 요소 중 일부는 내부에 다른 블록 레벨 요소를 포함할 수 있습니다. 대표적인 태그들을 아래 표로 정리했습니다.

블록 레벨 태그 (하위 블록 포함 가능)

태그 설명 하위 블록 요소 포함 가능 여부
<div> 일반적인 컨테이너 가능
<section> 문서의 구획을 나타냄 가능
<article> 독립적인 콘텐츠 단위 가능
<aside> 보조 콘텐츠 영역 가능
<header> 머리말 영역 가능
<footer> 꼬리말 영역 가능
<nav> 내비게이션 영역 가능
<form> 사용자 입력을 받는 폼 가능
<fieldset> 폼 요소 그룹화 가능
<li> 목록 항목 가능 (중첩 목록 포함 가능)
<td>, <th> 테이블 셀 가능 (셀 안에 블록 요소 포함 가능)
<body> 문서 본문 가능

인라인 태그 (하위 블록 포함 가능)

태그 설명 하위 블록 요소 포함 가능 여부
<a> 하이퍼링크(anchor). HTML5부터 블록 요소 포함 가능 가능
