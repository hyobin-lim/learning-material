웹 폰트 연결 방법 종합 가이드

1) <link> 방식 (가장 직관적)
HTML의 head에 외부 스타일시트 링크를 추가해 폰트를 로드합니다. 빠르고 간단하며, 대부분의 CDN(예: Google Fonts)에서 권장합니다. 
<head>
  <link href="https://fonts.googleapis.com/css2?family=Noto+Sans:wght@400;700&display=swap" rel="stylesheet">
</head>
<style>
  body { font-family: 'Noto Sans', sans-serif; }
</style>
이 문장은 <link> 방식으로 로드한 Noto Sans 폰트로 렌더링됩니다.
tip: display=swap을 사용하면 로딩 중 시스템 폰트로 먼저 표시되어 FOUT(깜빡임)이 줄어듭니다.
2) @import 방식 (CSS 내부에서 로드)
CSS 파일(또는 <style> 블록) 안에서 외부 폰트를 불러옵니다. 유지보수를 CSS에 집중할 때 유용합니다. 
/* CSS 파일 또는 <style> 블록 상단에 */
@import url('https://fonts.googleapis.com/css2?family=Indie+Flower&display=swap');

.title {
  font-family: 'Indie Flower', cursive;
}
이 문장은 @import 방식으로 로드한 Indie Flower 폰트로 렌더링됩니다.
tip: @import는 네트워크 요청이 연쇄적으로 발생할 수 있어 성능상 <link>보다 느릴 수 있습니다.
3) @font-face 방식 (원격/자체 호스팅 파일 직접 연결)
서버나 CDN에 올려둔 폰트 파일(woff2 권장)을 직접 연결합니다. 라이선스 관리나 커스텀 폰트 배포에 적합합니다. 
@font-face {
  font-family: 'MyInter';
  font-style: normal;
  font-weight: 400;
  font-display: swap;
  src: url('/fonts/Inter-Regular.woff2') format('woff2'),
       url('/fonts/Inter-Regular.woff') format('woff');
}
body { font-family: 'MyInter', sans-serif; }
이 문장은 @font-face로 연결한 Inter Regular (원격 파일)로 렌더링됩니다.
tip: woff2 사용을 권장합니다. 용량이 작고 대부분 브라우저에서 지원됩니다.
4) 로컬 폰트 우선 + 원격 폴백
사용자의 시스템에 폰트가 이미 있다면 로컬을 우선 사용하고, 없을 경우 원격 파일을 내려받게 합니다. 
@font-face {
  font-family: 'LocalFirstInter';
  font-style: normal;
  font-weight: 700;
  font-display: swap;
  src: local('Inter Bold'), local('Inter-Bold'),
       url('/fonts/Inter-Bold.woff2') format('woff2');
}
h1 { font-family: 'LocalFirstInter', system-ui, sans-serif; }
이 문장은 로컬에 Inter Bold가 있으면 로컬을, 없으면 원격 파일을 사용합니다.
tip: 로컬 폰트 이름은 OS별로 다를 수 있어 정확한 네이밍이 중요합니다.
5) 가변 폰트(Variable Font) 연결
한 파일로 여러 굵기/너비/기울임을 지원합니다. 파일 수를 줄이고 디자인 유연성을 높입니다. 
@font-face {
  font-family: 'InterVar';
  font-weight: 100 900; /* 지원하는 굵기 범위 */
  font-display: swap;
  src: url('/fonts/InterVariable.woff2') format('woff2');
}
.text { font-family: 'InterVar', system-ui, sans-serif; }
이 문장은 Inter Variable 폰트로 표시됩니다 (가변 굵기).
400
tip: 가변 폰트는 CSS에서 font-weight: 100 900처럼 범위를 선언합니다.
실무 팁 요약
  • 파일 포맷: woff2 우선, 필요한 경우 woff 폴백.
  • 성능: font-display: swap으로 로딩 중 텍스트 가독성 확보.
  • 캐싱: 자체 호스팅 시 적절한 캐시 헤더 설정(immutable, long max-age).
  • 문자 범위: unicode-range로 서브셋을 나누면 용량 절감.
  • 보안/라이선스: 상용 폰트는 웹 임베드 허용 범위와 도메인 제한(CORS)을 확인.

요약

- 가장 간단한 연결: <link> (권장)
- CSS 내부 관리: @import
- 커스텀/자체 호스팅: @font-face
- 최적화: local() 우선, woff2, font-display, unicode-range, variable font 활용
