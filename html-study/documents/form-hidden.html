Form 섹션 8 - 숨겨진 입력

전체 설명

숨겨진 입력(Hidden Input)은 화면에는 표시되지 않지만, 폼이 제출될 때 서버로 함께 전송되는 데이터를 담는 요소입니다.
사용자는 직접 수정할 수 없지만, 브라우저 개발자 도구에서는 확인할 수 있으므로 민감한 값은 반드시 서버에서 검증해야 합니다.

상세 설명

• <input type="hidden">은 UI에 나타나지 않지만 서버로 전송됩니다.
• 주로 CSRF 토큰, 추적 파라미터, 고정된 컨텍스트 정보(예: 사용자 ID, 세션 값)를 전달할 때 사용합니다.
• 숨겨진 입력은 클라이언트에서 쉽게 노출·변경될 수 있으므로 보안 검증은 서버에서 처리해야 합니다.

CSRF란?
CSRF(Cross-Site Request Forgery, 사이트 간 요청 위조)는 사용자가 의도하지 않은 요청을 공격자가 대신 보내도록 속이는 웹 공격 기법입니다.
예를 들어 로그인된 사용자가 악성 사이트를 방문했을 때, 그 사이트가 사용자의 권한으로 은행 송금 요청을 보내버릴 수 있습니다.
이를 막기 위해 서버는 CSRF 토큰을 발급하고, 폼에 숨겨진 입력으로 함께 전송해 요청이 정상적인 사용자의 것인지 확인합니다.
⚠️ 주의: 숨겨진 입력은 보안 수단이 아니라 단순 데이터 전달용입니다.
중요한 값은 서버에서 다시 확인하고 검증해야 합니다.

속성별 정리

속성설명예제 코드데모
type="hidden" 숨겨진 입력 지정 
<input type="hidden" name="csrf" value="token123">
 (화면에 표시되지 않음)
name 서버로 전송될 데이터의 키 
<input type="hidden" name="userId" value="42">
 (화면에 표시되지 않음)
value 서버로 전송될 실제 값 
<input type="hidden" name="role" value="admin">
 (화면에 표시되지 않음)
