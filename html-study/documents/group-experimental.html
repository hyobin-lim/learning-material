실험적 속성 그룹

아래 속성들은 최신 CSS 애니메이션 기능으로 애니메이션의 시작 지연, 스크롤 기반 실행, 실행 범위 지정 등 세부 동작을 제어하는 속성들입니다. 일부 브라우저에서만 제한적으로 지원됩니다.

animation-delay

애니메이션 시작 전 대기 시간을 지정합니다. 기본값은 0s이며, 단위는 초(s) 또는 밀리초(ms)를 사용할 수 있습니다.

  • 양수 값: 지정한 시간만큼 기다린 뒤 애니메이션 시작
    예) animation-delay: 2s; → 2초 후 실행
  • 음수 값: 애니메이션이 이미 진행된 상태에서 시작
    예) animation-delay: -1s; → 애니메이션이 1초 진행된 지점부터 시작
  • 복수 애니메이션: 여러 애니메이션을 쉼표로 나열할 경우 각각의 delay가 독립적으로 적용됨

활용 팁: 요소가 화면에 등장할 때 순차적으로 애니메이션을 실행하고 싶을 때 delay를 다르게 주면 자연스러운 연출이 가능합니다.

animation-composition

여러 애니메이션이 동일한 속성을 변경할 때 합성 방식을 지정합니다.

  • replace: 마지막 애니메이션이 이전 효과를 덮어씀
  • add: 여러 애니메이션 효과를 합산
  • accumulate: 반복 시 변화량을 누적

브라우저 지원 현황: Chrome, Edge, Safari, Firefox 최신 버전에서 점진적으로 지원. 구형 브라우저는 미지원.

주의사항: 실무 적용 시 호환성 확인 필요. 구형 환경에서는 fallback 스타일을 준비하는 것이 안전합니다.

animation-range

애니메이션을 특정 구간만 실행하도록 범위를 제한합니다.

  • animation-range: 25% 75%; → 전체 진행 중 중간 25%~75% 구간만 실행
  • animation-range: cover; → 전체 범위를 커버

브라우저 지원 현황: Chrome 일부 버전에서만 제한적으로 지원. Firefox, Safari, Edge 등은 대부분 미지원.

주의사항: 실험적 기술이므로 프로덕션 환경에서는 사용을 권장하지 않으며, 테스트나 데모 용도로만 활용하는 것이 좋습니다.

animation-timeline

기본 문서 타임라인 대신 사용자 정의 타임라인(예: ScrollTimeline, ViewTimeline)에 연결합니다.

  • auto: 기본 문서 타임라인
  • scroll(): 스크롤 진행에 따라 애니메이션 실행
  • view(): 뷰포트 진입/노출에 따라 실행

브라우저 지원 현황: 일부 최신 Chrome 계열 브라우저에서만 지원. Safari, Firefox, Edge 등은 대부분 미지원.

주의사항: 브라우저별 구현 차이가 크므로 실제 서비스보다는 실험적 프로젝트나 학습 목적에 적합합니다.
