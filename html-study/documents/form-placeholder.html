Form 섹션 9 - 자동완성 및 Placeholder

전체 설명

입력 필드에서는 사용자가 더 쉽게 작성할 수 있도록 다양한 보조 속성을 제공합니다.
이 속성들은 입력 힌트를 제공하거나, 브라우저의 자동완성 기능을 제어하거나, 맞춤법 검사 및 모바일 키보드 모드 지정 등 사용자 경험을 향상시키는 역할을 합니다.

속성 설명

• placeholder (힌트 텍스트): 입력 전 필드 안에 안내 문구를 표시해 사용자가 어떤 값을 입력해야 하는지 알려줍니다.
• autocomplete (자동완성): 브라우저가 이전 입력값을 제안할지 여부를 제어합니다. 로그인 폼이나 민감 정보 입력 시 꺼두는 경우가 많습니다.
• spellcheck (맞춤법 검사): 텍스트 입력 시 맞춤법 검사를 적용할지 여부를 지정합니다.
• inputmode (입력 모드): 모바일 환경에서 어떤 키보드 레이아웃을 띄울지 지정합니다. 예: 숫자 입력 시 숫자 키패드.
• readonly (읽기 전용): 입력값을 수정할 수 없게 하지만 서버로는 전송됩니다.

spellcheck가 텍스트 입력에만 의미가 있는 이유
맞춤법 검사는 “글자”를 대상으로 동작합니다. 따라서 <textarea>나 <input type="text">처럼 글자를 입력하는 필드에서는 맞춤법 검사가 적용됩니다.
하지만 <input type="number">나 <input type="date">처럼 숫자나 날짜를 입력하는 필드에는 글자가 없기 때문에 맞춤법 검사가 동작하지 않습니다.
즉, spellcheck 속성은 오직 텍스트 기반 입력 필드에서만 의미가 있습니다.

속성별 정리

속성 설명 예제 코드 데모
placeholder 힌트 텍스트 제공 
<input placeholder="이름을 입력하세요">
autocomplete 자동완성 on/off 
<input autocomplete="off">
spellcheck 맞춤법 검사 적용 여부 (텍스트 입력 필드에만 의미) 
<textarea spellcheck="true"></textarea>
inputmode 모바일 키보드 레이아웃 지정 
<input inputmode="numeric" placeholder="숫자만 입력">
readonly 읽기 전용 입력 (서버로는 전송됨) 
<input readonly value="수정 불가">
