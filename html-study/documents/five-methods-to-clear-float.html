float 해제(클리어) 5가지 방법

clear:both, clearfix, overflow, 부모 높이 고정(권장되지 않음), Flex/Grid — 각각의 동작 원리와 코드 데모

기본 개념 요약

float를 사용하면 요소가 일반 문서 흐름에서 빠져 부모의 높이가 0처럼 보이거나 다음 콘텐츠가 겹칠 수 있습니다. 이를 해결하는 대표적인 방법 다섯 가지를 실제 코드와 함께 비교합니다.

문제: 부모 높이 붕괴 목표: 부모가 자식 float을 포함 주의: 부작용(스크롤/잘림/유지보수성)

공통 예시 레이아웃

아래 박스 두 개를 좌우로 float시키고, 뒤에 일반 콘텐츠가 이어지는 상황을 가정합니다.

HTML 구조

<div class="container">
  <div class="float-left">왼쪽 float</div>
  <div class="float-right">오른쪽 float</div>
  <div class="content">다음 일반 콘텐츠</div>
</div>
파랑: 왼쪽 float
핑크: 오른쪽 float
노랑: 일반 콘텐츠

1) clear: both

float 뒤에 clear: both를 가진 요소를 하나 추가해 물리적으로 흐름을 끊어 부모가 높이를 갖도록 합니다. 간단하고 직관적이지만, 마크업에 별도의 ‘클리어 요소’를 넣어야 합니다.

데모

왼쪽 float
오른쪽 float
clear 요소 덕분에 부모가 float을 포함합니다.

코드

<style>
  .clear { clear: both; }
</style>

<div class="container">
  <div class="float-left">왼쪽 float</div>
  <div class="float-right">오른쪽 float</div>
  <!-- 흐름을 끊는 전용 요소 -->
  <div class="clear"></div>

  <div class="content">부모가 정상 높이를 가집니다.</div>
</div>

2) clearfix (실무에서 많이 쓰는 방식)

부모에 가상 요소(::after)를 만들어 clear: both를 적용하는 기법입니다. 마크업은 깔끔하게 유지하고, CSS만으로 float을 포함할 수 있어 재사용성이 높습니다.

데모

왼쪽 float
오른쪽 float
부모(.clearfix)가 float을 자동으로 포함합니다.

코드

<style>
  .clearfix::after {
    content: "";
    display: block;
    clear: both;
  }
</style>

<div class="container clearfix">
  <div class="float-left">왼쪽 float</div>
  <div class="float-right">오른쪽 float</div>
  <div class="content">마크업에 추가 요소가 필요 없습니다.</div>
</div>

3) overflow: auto 또는 hidden

부모에 overflow를 주면 새로운 블록 포맷팅 컨텍스트(BFC)가 생겨 float을 포함합니다. auto는 넘치면 스크롤이 생길 수 있고, hidden은 넘치는 내용을 잘라낼 수 있어 부작용을 고려해야 합니다.

데모 (overflow: auto)

왼쪽 float
오른쪽 float
부모가 float을 포함하지만, 내용이 넘치면 스크롤이 생길 수 있습니다.

코드 (overflow: auto)

<style>
  .overflow-auto { overflow: auto; }
</style>

<div class="container overflow-auto">
  <div class="float-left">왼쪽 float</div>
  <div class="float-right">오른쪽 float</div>
  <div class="content">BFC로 인해 부모가 포함됩니다.</div>
</div>

데모 (overflow: hidden)

왼쪽 float
오른쪽 float
넘치는 영역은 잘릴 수 있으니 주의하세요.

코드 (overflow: hidden)

<style>
  .overflow-hidden { overflow: hidden; }
</style>

<div class="container overflow-hidden">
  <div class="float-left">왼쪽 float</div>
  <div class="float-right">오른쪽 float</div>
  <div class="content">잘림 부작용을 고려해야 합니다.</div>
</div>

4) 부모 높이 고정 (권장되지 않음)

부모에 고정 높이를 주면 겉보기엔 포함된 것처럼 보이지만, 콘텐츠가 늘거나 줄 때 쉽게 깨집니다. 반응형 레이아웃, 동적 데이터, 다국어 등 변화에 취약하므로 일반적으로 권장되지 않습니다.

데모

왼쪽 float
오른쪽 float
콘텐츠가 늘면 잘리거나 겹칠 위험이 있습니다.

코드

<style>
  .fixed-height { height: 120px; overflow: hidden; }
</style>

<div class="container fixed-height">
  <div class="float-left">왼쪽 float</div>
  <div class="float-right">오른쪽 float</div>
  <div class="content">유지보수 측면에서 비추입니다.</div>
</div>

5) Flex와 Grid (최신 레이아웃)

float을 사용하지 않고 부모를 Flex나 Grid로 바꾸면 본질적으로 ‘클리어’가 필요 없습니다. 정렬, 간격, 반응형 배치를 더 쉽게 제어할 수 있어 새로운 프로젝트에서는 이 방식을 권장합니다.

Flex 데모

왼쪽 역할
오른쪽 역할
float 없이도 자연스럽게 레이아웃이 성립합니다.

Flex 코드

<style>
  .flex-container { display: flex; gap: 8px; flex-wrap: wrap; }
  .flex-item { flex: 1 1 240px; }
</style>

<div class="container flex-container">
  <div class="flex-item">왼쪽 역할</div>
  <div class="flex-item">오른쪽 역할</div>
</div>

Grid 데모

왼쪽 역할
오른쪽 역할
컬럼 제어와 반응형 배치가 유연합니다.

Grid 코드

<style>
  .grid-container {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));
    gap: 8px;
  }
</style>

<div class="container grid-container">
  <div class="grid-item">왼쪽 역할</div>
  <div class="grid-item">오른쪽 역할</div>
</div>

언제 무엇을 쓰면 좋을까

  • 간단한 페이지에서 빠르게 해결: clear 요소 또는 overflow 사용이 즉각적입니다.
  • 실무 유지보수와 재사용: clearfix 클래스를 만들어 공통으로 적용하세요.
  • 콘텐츠가 동적/반응형: 고정 높이는 피하고 overflow의 부작용도 점검하세요.
  • 새로운 프로젝트/현대적 레이아웃: Flex/Grid로 float 자체를 대체하는 것이 장기적으로 최선입니다.

빠른 비교 표

방법 장점 주의점 추천 상황
clear: both 단순, 직관적 마크업에 추가 요소 필요 소규모, 빠른 수습
clearfix 마크업 깔끔, 재사용 용이 CSS 규칙 관리 필요 실무 공통 유틸
overflow CSS 한 줄로 해결 스크롤/잘림 부작용 간단한 섹션
부모 높이 고정 겉보기 즉시 정리 콘텐츠 변화에 취약(비권장) 매우 제한적 특수 케이스
Flex/Grid 현대적, 반응형에 강함 기존 float 설계와의 호환 고려 새 프로젝트/리팩터링
이 문서는 실습을 위한 단일 HTML 파일입니다. 그대로 저장 후 브라우저에서 열면 각 데모를 확인할 수 있습니다.
