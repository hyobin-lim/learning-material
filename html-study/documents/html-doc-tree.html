HTML 문서 구조 트리 설명서

HTML5 문서의 계층(트리) 구조를 도식화해 보여주고, 접근성과 SEO 모범 사례, 흔한 오류 교정까지 정리했습니다.

1. 기본 트리 구조 도식

HTML 문서는 루트 <html> 요소 아래에 <head>와 <body>가 있는 계층 구조로 이루어집니다. 아래는 가장 기본적인 트리입니다.

<!DOCTYPE html>
html (lang="ko")
│
├── head
│   ├── meta (charset="UTF-8")
│   ├── meta (name="viewport" content="width=device-width, initial-scale=1")
│   ├── title
│   └── link (rel="stylesheet", href="css/structure.css")
│
└── body
    ├── header (머리글: 로고, 제목, 탐색)
    ├── nav (글로벌 내비게이션)
    ├── main (주요 콘텐츠, 문서당 1개 권장)
    │   ├── section (주제 블록)
    │   └── section (주제 블록)
    └── footer (바닥글: 회사/정책/사이트맵)
Tip: <main>은 페이지의 핵심 콘텐츠를 나타내며 문서당 하나만 사용하는 것이 좋습니다. 탐색이나 바닥글은 <main> 바깥에 위치합니다.

2. 예시 페이지 트리 (Dream Jeju)

다음은 “Dream Jeju” 페이지 구조를 트리로 상세히 풀어쓴 것입니다. 의미와 모범 사례를 함께 주석처럼 병기했습니다.

모범 사례 보충:
  • 이미지: 의미 있는 이미지는 alt를 제공하고, 장식 이미지는 alt=""로 표시합니다.
  • 내비게이션: <nav aria-label="주요 메뉴">처럼 목적을 설명하는 ARIA 라벨을 추가합니다.
  • 제목 구조: <h1>은 페이지 핵심 제목으로 1회 사용, 섹션에는 h2~h6로 계층을 유지합니다.
  • 간격/구분: <br>로 간격을 만들지 말고 CSS margin/padding을 사용합니다.
  • 성능: 이미지에 loading="lazy", 필요 시 width/height 속성으로 레이아웃 안정화.

3. 흔한 오류와 교정

잘못된 중첩: <b> 안에 <p>를 넣는 것은 비시맨틱하며 접근성에 불리합니다. 교정: 문장 강조는 <strong> 또는 <em>로, 문단은 독립 <p>로 구성하세요.
중복/누락 태그: 말미의 중복 </ul>은 제거하고, 모든 리스트는 <ul> 안에 <li>만 위치하도록 유지하세요.
의미 없는 링크: href="#"는 스크롤 점프와 접근성 문제를 야기할 수 있습니다. 교정: 실제 목적지(예: index.html 또는 "/")를 지정하거나 버튼 역할이면 <button>을 사용하세요.
오탈자 ID: id="headling" 대신 id="healing-section"처럼 의미가 명확한 이름을 사용하세요.

4. 접근성/SEO 체크리스트

  • 문서 선언/언어: DOCTYPE 존재, html lang 지정.
  • 인코딩/뷰포트: meta charset, meta viewport 설정.
  • 제목 계층: h1 1회, 섹션별 h2~h6 일관성 유지.
  • 이미지 대체 텍스트: 의미 이미지에 적절한 alt 제공.
  • 내비 라벨: aria-label로 탐색 목적 전달.
  • 링크 텍스트: “여기” 대신 목적이 드러나는 문구 사용.
  • 간격 제어: 과도한 <br> 금지, CSS로 레이아웃.
  • 성능: lazy loading, 이미지 최적화, 외부 리소스 최소화.

5. 트리 도식 + 코드 예제 (교정/보완 반영)

아래 코드는 위 트리 도식을 실제 HTML로 반영한 간단 샘플입니다.

<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Dream Jeju</title>
  <link rel="stylesheet" href="css/structure.css">
</head>
<body>
  <ul class="skip-links">
    <li><a href="#top">위로 가기</a></li>
    <li><a href="#main-content">본문 바로가기</a></li>
  </ul>

  <div id="container">
    <header id="top">
      <div id="logo">
        <a href="documents/index.html" aria-label="Dream Jeju 홈으로 이동">
          <h1>Dream Jeju</h1>
        </a>
      </div>
      <nav aria-label="주요 메뉴">
        <ul id="topMenu">
          <li><a href="#group-tours">단체 여행</a></li>
          <li><a href="#custom-tours">맞춤 여행</a></li>
          <li><a href="#gallery">갤러리</a></li>
          <li><a href="#contact">문의하기</a></li>
        </ul>
      </nav>
    </header>

    <hr aria-hidden="true">

    <main id="main-content" class="contents">
      <section id="healing-section">
        <h2>몸과 마음이 치유되는 섬</h2>
        <div class="detail">
          <img src="../images/healing.jpg" alt="제주의 푸른 바다와 오름 풍경" loading="lazy">
          <p><strong>쉼(Healing)</strong>의 공간으로 안내합니다.</p>
          <p>탁 트인 바다와 시원한 바람을 느끼며 오름을 오르고, 올레길을 걷다 보면 온전히 나에게 집중할 수 있습니다.</p>
        </div>
        <div class="schedule">
          <h3>상세 일정</h3>
          <ul>
            <li>여행 기간: 2박 3일</li>
            <li>여행 일정: 상담을 통해 결정 및 조정 가능합니다.</li>
          </ul>
        </div>
      </section>

      <section id="activity-section">
        <h2>다양한 액티비티가 기다리는 섬</h2>
        <div class="detail">
          <img src="../images/activity.jpg" alt="패러글라이딩, 스쿠버다이빙, 전동바이크 등 제주 액티비티" loading="lazy">
          <p><strong>모험과 스릴</strong>이 넘치는 레저의 천국으로 안내합니다.</p>
          <p>둘러보기만 하는 여행을 하셨나요? 패러글라이딩, 스킨스쿠버, 전동바이크 등 다양한 활동을 즐겨보세요.</p>
          <p>요트 체험과 배낚시도 빼놓을 수 없죠.</p>
        </div>
      </section>
    </main>

    <footer>
      <section id="bottomMenu" aria-label="바닥글 메뉴">
        <ul>
          <li><a href="#about">회사 소개</a></li>
          <li><a href="#privacy">개인정보처리방침</a></li>
          <li><a href="#terms">여행약관</a></li>
          <li><a href="#sitemap">사이트맵</a></li>
        </ul>
      </section>
    </footer>
  </div>
</body>
</html>

이 문서는 학습과 실무 적용을 위해 작성되었습니다. 필요하면 트리 도식을 프로젝트 가이드로 복사해 사용하세요.
