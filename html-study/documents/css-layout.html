CSS 레이아웃 완전 정리

1. CSS에서의 레이아웃이란?

CSS 코드 관점에서의 레이아웃(Layout)은 웹 페이지의 요소들을 화면에 어떻게 배치하고, 크기를 계산하며, 서로의 관계를 정의하는지를 결정하는 규칙과 속성들의 집합입니다.
즉, HTML로 구조를 만들고 CSS로 그 구조를 어떻게 보이게 할지를 설계하는 것입니다.

2. 전체적인 큰 틀

  • CSS 레이아웃은 요소를 화면에 어떻게 배치할지 결정하는 규칙
  • 박스 모델이 기본, 그 위에 display → position → flex/grid → media query로 확장
  • 최신 레이아웃은 Flexbox와 Grid 중심으로 설계, 구식 방식(Float, Table)은 보조적으로 사용
  • 레이아웃은 단순히 위치만이 아니라 크기 계산, 정렬, 흐름, 반응형 대응까지 포함

3. 주요 속성 그룹

그룹 대표 속성 설명
박스 모델(Box Model) margin, padding, border, width, height 박스 요소 크기와 간격
디스플레이(Display) display 요소의 기본 배치 방식 결정 (block, inline, flex, grid 등)
정렬과 흐름(Alignment & Flow) text-align, vertical-align, justify-content, align-items 요소 내부나 그룹의 정렬 제어 (text-align, justify-content 등)
위치(Positioning) position, top, left, z-index 요소를 문서 흐름에서 벗어나 좌표 기반 배치
흐름 제어 float, clear 요소를 좌·우로 띄워 문서 흐름 변경
레이아웃 시스템 flex(1차원-가로/세로), grid(2차원-행+열) 현대적 레이아웃 방식
반응형 레이아웃 @media, minmax, fr 화면 크기·장치 특성에 따라 레이아웃 변화
콘텐츠 넘침 처리 overflow 콘텐츠가 부모 영역을 벗어날 때 처리

5. 핵심 요약

  • Text-align: 블록 내부 인라인 콘텐츠 정렬
  • Block / Inline / Inline-block: 기본 배치 방식의 차이를 이해해야 함
  • Block: 줄 전체 차지, 줄바꿈 발생
  • Inline: 콘텐츠 크기만큼, 줄 안에서 배치
  • Inline-block: 줄 안에서 배치 + 크기 지정 가능
  • Flexbox: 1차원 레이아웃, 정렬과 크기 조절에 유연
  • Grid: 2차원 레이아웃, 복잡한 구조를 직관적으로 설계
  • Table: 행·열 구조, 과거 레이아웃 기법
  • Float: 요소를 좌·우로 띄워 흐름 변경, 구식이지만 특정 상황에서 사용
  • Position: 요소의 위치를 세밀하게 제어
  • Overflow: 넘치는 콘텐츠 처리 방식 지정

6. 추가 팁

  • 모바일 퍼스트: 작은 화면부터 레이아웃을 정의하고 점차 확장
  • Flexbox + Grid 혼합 사용: 큰 틀은 Grid로, 세부 정렬은 Flexbox로 처리하면 효율적
  • box-sizing: border-box를 활용하면 크기 계산이 쉬워짐
  • 브라우저 개발자 도구로 다양한 화면 크기에서 테스트
  • 접근성 고려: 레이아웃은 시각적 구조뿐 아니라 스크린리더 등 접근성에도 영향
  • 최신 기법 활용: Float, Table은 구식 → Flex와 Grid 중심으로 학습
