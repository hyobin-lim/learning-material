Form 섹션 4 - 레이블과 접근성

전체 설명

레이블과 접근성(Label & Accessibility)은 **사용자가 입력 필드를 쉽게 이해하고 사용할 수 있도록 돕는 요소와 속성**입니다.
<label> (레이블 태그)는 입력 필드와 연결되어 어떤 값을 입력해야 하는지 알려주고,
<fieldset> (입력 그룹 태그)과 <legend> (그룹 제목 태그)는 관련 입력들을 묶어 의미를 명확히 합니다.
또한 aria-describedby (접근성 설명 연결)와 role="alert" (경고 역할) 같은 접근성 속성은 스크린리더 사용자에게 추가 정보를 제공합니다.

상세 설명

각 입력은 <label>로 명확히 연결해야 합니다.
추가 설명이나 오류 메시지는 aria-describedby로 연결하면 스크린리더에 전달됩니다.
관련 입력은 <fieldset> + <legend>로 그룹화하여 의미를 부여합니다.

속성 중심

속성 중심은 태그에 붙는 속성 자체를 설명하는 관점입니다.
예를 들어 for, id, aria-label, aria-describedby, role="alert" 같은 속성이 어떤 역할을 하는지 보여줍니다.

속성한글 이름설명예제 코드데모
for 연결 대상 label이 어떤 입력과 연결되는지 지정 
<label for="username">아이디</label>
<input id="username" type="text">
id 식별자 입력 필드의 고유 식별자, label과 연결할 때 사용 
<input id="email" type="email">
aria-label 접근성 레이블 화면에 표시되지 않는 레이블을 제공 
<input type="text" aria-label="검색어 입력">
aria-describedby 접근성 설명 연결 입력 필드와 설명/오류 메시지를 연결 
<input id="pw" aria-describedby="pwHelp">
<small id="pwHelp">비밀번호는 8자 이상</small>
 비밀번호는 8자 이상
role="alert" 경고 역할 오류 메시지를 스크린리더에 즉시 알림 
<div role="alert">입력 오류 발생</div>
입력 오류 발생

태그 중심

태그 중심은 실제 태그 단위로 설명하는 관점입니다.
<label>, <fieldset>, <legend> 같은 태그를 사용할 때 속성을 어떻게 활용하는지와 브라우저에서 어떻게 보이는지를 보여줍니다.

label의 for 값은 연결할 입력의 id와 반드시 동일해야 합니다.
태그설명예제 코드데모
label (레이블) for-id 매칭으로 입력 필드와 연결 
<label for="email">이메일</label>
<input id="email" type="email">
aria-describedby (접근성 설명 연결) 설명/오류 텍스트를 입력과 연결 
<input id="u" aria-describedby="uHelp uErr">
<small id="uHelp">아이디는 4~20자</small>
<div id="uErr" role="alert" hidden>형식 오류</div>
 아이디는 4~20자
