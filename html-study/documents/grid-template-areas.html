CSS Grid: grid-template-areas 완전 정리

grid-template-areas는 그리드 컨테이너의 행(row)과 열(column)을 영역 이름(area)으로 시각적으로 배치하는 선언적 문법입니다. 각 자식 요소는 grid-area로 영역 이름에 매칭되어 배치됩니다.

핵심 개념

  • 문자열 행: 각 행을 큰따옴표로 감싸고, 공백으로 셀을 구분합니다.
  • 영역 이름: 동일 이름이 직사각형을 이루면 하나의 영역으로 합쳐집니다.
  • 빈 셀: 점(.)은 빈 공간을 의미합니다.
  • 아이템 매칭: 자식 요소에 grid-area: 이름;을 지정하면 해당 영역에 배치됩니다.
  • 반응형 배치: 미디어쿼리로 문자열 패턴을 바꾸어 재배치할 수 있습니다.

문법과 예시

.grid {
  display: grid;
  grid-template-columns: 200px 1fr 200px;  /* 열 크기 */
  grid-template-rows: auto 1fr auto;       /* 행 크기 */
  grid-template-areas:
    "header header header"
    "nav    main   sidebar"
    "footer footer footer";
}

.header  { grid-area: header; }
.nav     { grid-area: nav; }
.main    { grid-area: main; }
.sidebar { grid-area: sidebar; }
.footer  { grid-area: footer; }

상단 헤더, 좌측 내비, 본문, 우측 사이드바, 하단 푸터의 전형적 3열 레이아웃.

규칙과 주의사항

  • 직사각형 규칙: 같은 영역 이름은 반드시 직사각형을 이뤄야 합니다. 꺾이거나 L자 모양은 불가.
  • 행 길이 일치: 모든 문자열 행은 동일한 셀 수(열 수)를 가져야 합니다.
  • 중복 이름: 같은 이름을 여러 영역으로 나눠 쓸 수 없습니다. 하나의 이름은 하나의 직사각형만.
  • 빈 셀: .로 비워두면 해당 그리드 셀은 빈 상태로 남습니다(컨텐츠가 없으면 높이 0 가능).
  • 암시적 트랙: 영역 외의 아이템이나 크기 선언으로 암시적 행/열이 생성될 수 있습니다.

잘못된 vs 올바른 패턴

grid-template-areas:
  "a a ."
  "a . .";  /* a가 L자 모양 → 잘못된 패턴 */
a 영역이 직사각형이 아니므로 무효입니다. 
grid-template-areas:
  "a a ."
  "a a .";  /* a가 2x2 직사각형 → 올바름 */
동일 이름이 깔끔한 직사각형을 형성하면 유효합니다.

빈 셀과 영역 배치

grid-template-areas:
  "header header header"
  "nav    main   ."
  "footer footer footer";

우측 마지막 셀을 .로 비워 레이아웃의 여백/광고/추후 확장을 위한 자리로 둘 수 있습니다.

반응형 재배치 (media queries)

.grid {
  display: grid;
  grid-template-columns: 200px 1fr 200px;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header header header"
    "nav    main   sidebar"
    "footer footer footer";
}

/* 좁은 화면에서는 단일 열로 재배치 */
@media (max-width: 720px) {
  .grid {
    grid-template-columns: 1fr;
    grid-template-rows: auto auto 1fr auto auto;
    grid-template-areas:
      "header"
      "nav"
      "main"
      "sidebar"
      "footer";
  }
}

grid-template-areas와 트랙 크기 상호작용

grid-template-areas는 grid-template-rows, grid-template-columns와 함께 쓰이며, 영역 이름은 배치만 정의하고 크기는 트랙 지정(px, %, fr, minmax, auto)으로 결정합니다. 

.grid {
  display: grid;
  grid-template-columns: 240px 1fr minmax(200px, 320px);
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header header header"
    "nav    main   sidebar"
    "footer footer footer";
}

아이템 매칭과 자동 배치

/* 이름으로 직접 배치 */
.header  { grid-area: header; }
.main    { grid-area: main; }

/* 이름 없는 아이템은 auto-placement 규칙에 따라 남는 셀로 배치됨 */
.card { /* grid-area 미지정 → 빈 셀에 자동 배치 */ }

베스트 프랙티스

  • 의미 있는 이름: header, main, sidebar처럼 역할 기반으로 이름을 정합니다.
  • 직사각형 보장: 디자인 변경 시에도 영역이 직사각형을 유지하도록 문자열을 점검합니다.
  • 미디어쿼리 전략: 데스크톱/태블릿/모바일 별로 문자열을 준비해 빠르게 재배치합니다.
  • 트랙 크기와 분리: 배치(areas)와 크기(rows/columns)를 분리해 유지보수성을 높입니다.
  • 접근성 고려: 시각적 순서가 바뀌어도 DOM 순서는 논리적 흐름을 유지하세요.

빠른 체크리스트

  • 행의 열 수 동일? 모든 문자열 행의 셀 개수가 같은가.
  • 영역이 직사각형? 같은 이름이 이어져 직사각형을 이루는가.
  • 빈 셀은 점으로? 비워둘 자리는 .로 명시했는가.
  • 아이템에 grid-area? 각 자식에 올바른 grid-area를 지정했는가.
  • 크기 지정 분리? 행·열 크기는 grid-template-rows/columns로 관리하는가.
