CSS 박스(box) 요소 핵심 정리

모든 HTML 요소는 브라우저에서 하나의 박스(Box)로 표현됩니다.
이 박스는 콘텐츠(content), 패딩(padding), 테두리(border), 마진(margin) 네 가지 영역으로 구성되며,
이를 합쳐서 CSS 박스 모델(Box Model)이라고 부릅니다.

CSS 박스 모델은 웹 레이아웃의 뼈대입니다.

  • width/height로 크기를 정하고,
  • padding으로 안쪽 여백을 확보하며,
  • margin으로 바깥 간격을 조절하고,
  • box-sizing으로 크기 계산 방식을 통제합니다.
이 네 가지를 이해하면 요소의 크기와 간격을 정확히 제어할 수 있고, 레이아웃을 안정적으로 설계할 수 있습니다.

속성 설명 예제 코드 데모 비고
width
  • 박스의 가로 크기입니다.
  • 기본값은 auto로, 부모 컨테이너나 콘텐츠 영역에 따라 달라집니다.(블록)
  • 인라인 요소에는 width가 적용되지 않고, 블록이나 인라인-블록 요소에서만 의미가 있습니다.
 
.block { display: block; width: 200px; }
.inline-block { display: inline-block; width: 200px; }
.inline { display: inline; width: 200px; /* 효과 없음 */ }
블록: width 200px
인라인: width 적용되지 않음 인라인-블록: width 200px
인라인은 내용 길이에 따라 너비가 결정되고 width는 무시됨.
 블록은 가로 전체를 차지하지만 width 지정 시 그만큼만 사용.
height
  • 박스의 세로 크기입니다.
  • 기본값은 auto로, 부모 컨테이너나 콘텐츠 높이에 따라 달라집니다.
  • 인라인 요소에는 height가 적용되지 않고(라인 높이로 표시), 블록이나 인라인-블록 요소에서만 의미가 있습니다.
 
.block { height: 80px; }
.inline-block { display: inline-block; height: 80px; }
.inline { display: inline; height: 80px; /* 효과 제한적 */ }
블록: height 80px
인라인: height 영향 작음 인라인-블록: height 80px
인라인 요소는 height 대신 line-height 영향이 큼.
 이미지 등 일부 인라인-대체 요소는 height 적용.
padding
  • 콘텐츠와 테두리 사이의 안쪽 여백입니다.
  • 배경색과 배경 이미지는 패딩 영역까지 채워집니다.
  • box-sizing 설정에 따라 총 크기 계산 방식이 달라집니다.
 
.content-box { box-sizing: content-box; width: 200px; padding: 20px; }
.border-box { box-sizing: border-box; width: 200px; padding: 20px; }
content-box (총 너비 240px)
border-box (총 너비 200px)
content-box: width + padding + border. border-box: width 안에 padding/border 포함.
 텍스트 가독성 향상에 필수. 레이아웃 계산 주의.
margin
  • 박스 바깥쪽 여백입니다.
  • 인접한 블록 요소의 위·아래 마진은 겹쳐서 하나의 값으로 표시될 수 있습니다(마진 겹침).
  • 좌우 마진은 겹치지 않습니다.
 
.boxA { margin: 24px 0; }
.boxB { margin: 32px 0; }
/* 위아래 마진은 큰 값 하나로 겹침(collapse) */
박스 A (margin-top/bottom 24px)
박스 B (margin-top/bottom 32px)
A와 B 사이 간격은 24+32가 아닌 32로 표시됨(마진 겹침).
 좌/우 마진은 겹치지 않음. 인라인 마진은 좌우만 뚜렷함.
box-sizing
  • width/height가 포함하는 영역의 기준을 결정합니다.
  • content-box(기본): width/height는 콘텐츠 영역만 포함합니다. → padding과 border는 별도 계산
  • border-box: width/height 안에 padding과 border까지 포함합니다. → 레이아웃 관리가 편리 
/* 권장: 전체 문서에 border-box 적용 */
*, *::before, *::after { box-sizing: border-box; }
content-box
border-box
같은 설정에서도 border-box가 총 크기 관리가 쉬움.
 디자인 시스템에서 기본을 border-box로 통일하는 경우가 많음.
auto, min/max
  • auto: 상황(문맥)에 따라 크기가 자동 결정
  • min-width, max-width: 최소·최대 크기를 제한해 반응형 레이아웃에서 유용
  • height: auto: 이미지나 콘텐츠 비율을 유지할 때 자주 사용 
.card { width: 100%; max-width: 640px; min-width: 280px; height: auto; }
반응형 카드: 부모 너비에 맞추되 280~640px 사이로 제한
이미지 등 콘텐츠 비율 유지에 height:auto가 유용.
display에 따른 차이 display 값에 따라 width/height 적용 여부가 달라짐.
  • block: 가로 전체를 차지, width/height 적용 가능
  • inline: 줄 흐름을 유지, width/height 적용 제한적
  • inline-block: 줄 흐름 유지하면서도 width/height 적용 가능 → 버튼, 배지 등에 자주 사용 
.inline { display: inline; }      /* width/height 적용 제한 */
.inline-block { display: inline-block; } /* 적용됨 */
.block { display: block; }        /* 적용됨 */
inline inline-block
block
inline은 크기 지정이 제한적이며 줄 흐름을 유지.
 버튼/배지 등은 inline-block이 실용적.

추가 팁

  • 컨테이너 관리: 레이아웃 안정성을 위해 전체에 border-box를 적용하는 것이 일반적입니다.
  • 마진 겹침 회피: 부모에 padding을 주거나 overflow: auto/hidden을 설정하면 겹침을 피할 수 있습니다.
  • 패딩과 배경: 배경색/이미지는 padding 영역까지 채워집니다.
  • 반응형: 퍼센트 width + max-width 조합이 실전에서 자주 쓰입니다.
