Form 섹션 7 - 파일 업로드

전체 설명

파일 업로드(File Upload)는 사용자가 자신의 컴퓨터나 모바일 기기에서 파일을 선택해 서버로 전송할 수 있도록 하는 기능입니다.
<input type="file"> 태그를 사용하며, 폼에서는 반드시 enctype="multipart/form-data"를 설정해야 합니다.
파일 업로드 시 accept(허용 파일 유형), multiple(다중 선택), capture(캡처 장치 지정) 같은 속성을 활용해 입력 방식을 제어할 수 있습니다.

상세 설명

• accept (허용 파일 유형): 업로드 가능한 파일 형식을 제한 (예: image/*, .pdf, audio/*, video/*)
• multiple (다중 선택): 여러 파일을 한 번에 선택 가능 (PC에서는 Ctrl/Shift 키로 다중 선택)
• capture (캡처 장치 지정): 모바일 환경에서 카메라·마이크를 직접 호출해 파일 입력
• enctype="multipart/form-data" (폼 인코딩): 파일 업로드 시 반드시 필요한 인코딩 방식
• 파일 크기 제한: HTML 속성으로 직접 제한할 수 없으며, 서버 측에서 검증해야 함
• UI 커스터마이징: 기본 파일 입력 UI는 브라우저마다 다르며, CSS로 완전한 커스터마이징은 어려움

왜 반드시 enctype="multipart/form-data"를 설정해야 할까?
기본 폼 인코딩 방식(application/x-www-form-urlencoded)은 글자 데이터만 처리할 수 있습니다.
하지만 파일은 이미지·문서·음악처럼 덩어리 데이터(바이너리)라서 글자 방식으로 보내면 깨지거나 전송되지 않습니다.
multipart/form-data는 폼 데이터를 여러 조각으로 나누어 텍스트는 텍스트대로, 파일은 파일 그대로 포장해 서버로 보냅니다.
즉, 파일 업로드가 포함된 폼은 반드시 enctype="multipart/form-data"를 지정해야 서버가 파일을 원래 모습 그대로 받을 수 있습니다.

속성별 정리

항목설명예제 코드데모
file 입력 파일 선택 UI 제공 
<input type="file" accept="image/*" multiple>
폼 인코딩 파일 전송을 위한 설정 (반드시 multipart/form-data) 
<form enctype="multipart/form-data">...</form>
accept (허용 파일 유형) 특정 파일 형식만 선택 가능 
<input type="file" accept=".pdf,.docx">
<input type="file" accept="audio/*">
capture (캡처 장치 지정) 모바일에서 카메라·마이크 직접 호출 
<input type="file" accept="image/*" capture="camera">
