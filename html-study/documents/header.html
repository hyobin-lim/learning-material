<header> 태그 정리

페이지나 섹션의 머리말로, 제목·로고·탐색(내비게이션)·검색·유틸리티 링크·책의 표지와 같이 “첫인상과 맥락”을 제공하는 영역.

쉬운 설명 코드 예제 데모 화면 베스트 프랙티스 / 주의

의미: <header>는 문서 전체나 특정 섹션의 시작 부분(머리말)을 나타냅니다. 보통 <h1~h6> 제목, 로고, 내비게이션, 검색폼, 사용자 메뉴 등을 포함해 “여기가 무엇인지”를 알려줍니다.

사용 위치: 페이지 최상단의 사이트 헤더뿐만 아니라, <article>나 <section> 내부에도 각각의 헤더를 둘 수 있습니다. 즉, “문서 전체”와 “부분 문서” 모두에 적용 가능합니다.

중요 구분: 시각적 머리말과 의미론적 머리말을 혼동하지 않아야 합니다. 단지 꾸미는 상단 바가 아니라, 그 영역이 문서나 섹션의 정체성을 표현해야 합니다.

 
<header>
  <div class="brand">
    <span class="logo" aria-hidden="true"></span>
    <h1>My Site</h1>
  </div>

  <nav aria-label="주요 메뉴">
    <ul>
      <li><a href="#">홈</a></li>
      <li><a href="#">블로그</a></li>
      <li><a href="#">문의</a></li>
    </ul>
  </nav>

  <form action="/search" role="search">
    <label class="sr-only" for="q">검색어</label>
    <input id="q" name="q" type="search" placeholder="검색...">
  </form>
</header>

My Site

  • 홈
  • 블로그
  • 문의
  • 문서 내 다중 사용 가능: 페이지 전체 헤더 외에도 각 <article>·<section>에 개별 헤더를 둘 수 있습니다.
  • 헤딩 레벨 설계: 사이트 제목은 보통 최상단에서 <h1>를 쓰고, 섹션 내부 헤더는 해당 맥락에 맞는 <h2> 이하를 사용합니다.
  • 내비게이션 포함 여부: 헤더 안에 <nav>를 포함해도 되고, 별도로 둬도 됩니다. 의미적으로 “머리말에 탐색이 있다”면 포함해도 자연스럽습니다.
  • 접근성: 검색 폼엔 role="search" 또는 명확한 라벨을 제공하고, 아이콘엔 aria-hidden 처리로 스크린리더 잡음을 줄입니다.
