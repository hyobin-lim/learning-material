CSS transition 속성 정리

상태 변화(hover, focus, active 등)를 부드러운 애니메이션으로 표현하는 방법을 예제와 데모로 정리합니다.

1. transition 기본 개념

transition은 특정 CSS 속성이 바뀔 때, 그 변화를 지정한 시간과 속도 곡선에 맞춰 자연스럽게 이어지도록 합니다.

.box-basic {
  width: 100px;
  height: 100px;
  background-color: #93c5fd;
  transition: background-color 0.5s ease, transform 0.5s ease;
}
.box-basic:hover {
  background-color: #e74c3c;
  transform: scale(1.2);
}
hover
Tip: hover 시 색상과 크기가 동시에 바뀌어 transition의 기본 개념을 직관적으로 확인할 수 있습니다.

2. transition 관련 속성

속성설명예시
transition-property 애니메이션할 CSS 속성 지정
값: all, background-color, transform, opacity 등		 transition-property: transform;
transition-duration 변화가 진행되는 시간
값: 시간 단위 s(초), ms(밀리초)		 transition-duration: 2s;
transition-timing-function 속도 곡선 지정
값: ease, linear, ease-in, ease-out, ease-in-out, cubic-bezier()		 transition-timing-function: ease-in-out;
transition-delay 변화 시작 전 대기 시간
값: 시간 단위 s, ms		 transition-delay: 1s;
transition (축약형) 위 속성을 한 줄에 작성
순서: property(속성) → duration(지속시간) → timing-function(타이밍 함수) → delay(지연시간)		 transition: all 0.5s ease 0.2s;

축약형 설명: transition은 네 가지 속성을 순서대로 한 줄에 작성할 수 있습니다.
예를 들어 transition: all 0.5s ease 0.2s; 는 모든 속성을 0.5초 동안 ease 곡선으로 애니메이션하고, 0.2초 후에 시작한다는 의미입니다.

4. 타이밍 함수 비교

hover 시 점이 서로 다른 방향/효과로 움직입니다. 각 타이밍 함수에 따라 속도 곡선이 달라집니다.

ease (오른쪽)
linear (아래쪽)
ease-in (왼쪽)
ease-out (위쪽)
ease-in-out (확대)
cubic-bezier (회전)
Tip: cubic-bezier(x1, y1, x2, y2)로 커스텀 곡선을 만들면 이동뿐 아니라 회전·확대·축소 같은 다양한 움직임을 자연스럽게 제어할 수 있습니다.

5. 다양한 transition 효과

색상 변경, 크기 변경, 위치 이동, 그림자, 테두리 반경 등을 동시에 지정할 수 있습니다.

.combo {
  transition:
    background-color 400ms ease,
    transform 500ms ease-in-out,
    box-shadow 500ms ease,
    border-radius 400ms ease;
}
.combo:hover {
  background-color: #4338ca;
  transform: translateY(-40px) rotate(-15deg) scale(1.2);
  box-shadow: 0 16px 30px rgba(99,102,241,0.4);
  border-radius: 50%;
}
hover
Tip: 성능을 고려하면 transform과 opacity는 GPU 가속을 받아 부드럽고 비용이 적습니다. 레이아웃을 다시 계산하는 width, height, top, left 등의 transition은 과도하게 사용하지 않는 것이 좋습니다.

6. 요약

Transition 핵심 요약
항목 핵심 포인트
속성 선택 transition-property로 변화시킬 속성을 명확히 지정합니다.
시간 설계 duration은 컨텐츠 종류에 맞춰 적절히, delay는 과용하지 않습니다.
속도 곡선 의도에 맞춘 timing-function을 선택하거나 cubic-bezier로 커스텀합니다.
성능 transform, opacity 위주로, 레이아웃 변경 속성은 최소화합니다.
일관성 동일 UI 패턴에 같은 transition 값을 적용해 사용자의 예측 가능성을 높입니다.
