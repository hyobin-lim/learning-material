Form 섹션 1 - 기본 구조

개요

폼(Form, 폼 태그)은 사용자 입력을 서버로 전송하기 위한 컨테이너입니다.
<form> (폼 태그), <fieldset> (입력 그룹 태그), <legend> (그룹 제목 태그)를 사용하면 구조를 명확히 하고 접근성을 높일 수 있습니다.

<form> (폼 태그): 입력 전체를 감싸는 컨테이너
<fieldset> (입력 그룹 태그): 관련 입력들을 묶어줌
<legend> (그룹 제목 태그): 그룹에 제목을 붙여줌

중요 속성

  • action (액션): 데이터를 제출할 서버 주소(URL)
  • method (메서드): 전송 방식, GET(주소에 붙여서 전송) / POST(본문에 담아 전송)
  • enctype (인코딩 타입): 데이터 전송 형식 지정. 파일 업로드 시 multipart/form-data (멀티파트/폼 데이터) 사용
  • novalidate (검증 비활성화): 브라우저 기본 입력 검증을 끔
  • autocomplete (자동완성): 브라우저 자동완성 기능 켜기/끄기
  • target (응답 표시 위치): 제출 후 응답을 표시할 창이나 프레임 지정
multipart/form-data (멀티파트/폼 데이터):
파일 업로드 시 반드시 지정해야 하는 전송 방식입니다. 텍스트와 파일을 함께 서버로 보낼 수 있도록 데이터를 여러 부분으로 나누어 전송합니다.
보안상 CSRF 방지를 위해 서버 검증과 토큰 사용을 권장합니다.
항목설명예제 코드데모
<form> 기본 action, method, autocomplete 설정 
<form action="/submit" method="post" autocomplete="on">
  이름: <input type="text" name="username" required>
  <button type="submit">전송</button>
</form>
이름:
<fieldset> & <legend> 관련 입력 그룹화 및 제목 제공 
<fieldset>
  <legend>로그인</legend>
  아이디: <input type="text" name="id">
  비밀번호: <input type="password" name="pwd">
</fieldset>
