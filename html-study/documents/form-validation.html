Form 섹션 6 - 검증

전체 설명

검증(Validation)은 사용자가 입력한 값이 올바른지 확인하는 과정입니다.
HTML5는 기본 검증 기능을 제공하며, 자바스크립트 없이도 필수 입력, 정규식 패턴, 숫자/날짜 범위 등을 지정할 수 있습니다.
브라우저는 입력값의 유효성 상태에 따라 :valid / :invalid CSS를 적용해 시각적으로 표시합니다.
검증은 폼 제출 시점에 실행되며, 조건에 맞지 않으면 브라우저가 기본 오류 메시지를 보여줍니다.

속성별 정리

속성 용도 주의/메모 예제 코드
required 필수 입력 지정 빈 값일 경우 제출 불가. 체크박스/라디오에도 적용됨. 
<input required>
pattern 정규식으로 형식 제한 type이 text/tel/email 등 텍스트 계열일 때 사용. 전체 매칭 기준(암묵적 ^/$). 
<input type="tel" pattern="^010-[0-9]{4}-[0-9]{4}$">
min / max 최소/최대 범위 지정 number, date, time 등 범위형 타입에서 동작. 값 비교는 타입에 따라 숫자/날짜 기준. 
<input type="number" min="1" max="10">
step 증가/감소 단위 지정 number/range 등에서 증감 간격 정의. step="any"로 자유 입력 허용. 
<input type="number" step="0.5">
minlength / maxlength 문자 길이 제한 텍스트 계열(type="text","email","password" 등)과 textarea에 적용. 
<input type="text" minlength="3" maxlength="10">
type="email" / "url" 형식 자동 검증 브라우저가 RFC 기반의 기본 형식 체크 수행. placeholder와 혼동 금지. 
<input type="email" placeholder="you@example.com">
:valid / :invalid 유효성 상태 CSS 현재 값이 제약을 만족/불만족할 때 스타일링. 실시간 피드백에 유용. 
input:invalid { border-color:#e74c3c }
input:valid { border-color:#2ecc71 }
novalidate 폼 검증 비활성화 폼 제출 시 기본 검증을 건너뜀. 서버 검증만 사용할 때 유용. 
<form novalidate>...</form>
formnovalidate 버튼 단위로 검증 끄기 특정 제출 버튼에서만 검증 우회. 
<button type="submit" formnovalidate>검증 없이 제출</button>
title 힌트/오류 메시지 보조 pattern과 함께 사용해 기대 형식을 설명(기본 메시지에 더해 표시). 
<input pattern="[A-Z]{3}" title="대문자 3자리">
