CSS 단위 정리 (계산 예시 포함)

브라우저 기본값

대부분의 브라우저는 html { font-size: 16px; }를 기본으로 합니다.
따라서 1rem = 16px, 2rem = 32px
1em = 현재 요소의 font-size (부모가 20px이면 1em = 20px)
pt는 인쇄 기준이지만 화면에서는 1pt ≈ 1.333px로 렌더링됩니다.

절대 단위

단위설명코드데모
px 픽셀 단위. 화면에서 항상 동일한 크기를 유지합니다.
장점: 예측 가능, 정확한 레이아웃 제어 가능
주의: 접근성 확대 시 유연성이 떨어짐
예시 계산: width:100px → 항상 100픽셀 
font-size:16px;
텍스트 16px
pt 포인트. 인쇄 매체 기준 단위.
장점: 인쇄물과 동일한 크기 표현 가능
주의: 화면에서는 1pt ≈ 1.333px로 변환됨
예시 계산: 12pt ≈ 16px 
font-size:12pt;
텍스트 12pt
cm 센티미터. 실제 물리적 길이.
장점: 인쇄용 문서에서 정확한 크기 지정
예시 계산: 1cm ≈ 37.8px 
width:1cm;
박스 1cm
mm 밀리미터. 인쇄용 단위.
예시 계산: 10mm ≈ 37.8px 
width:10mm;
박스 10mm
in 인치. 1in = 96px.
장점: 인쇄물과 동일한 크기 표현 가능
예시 계산: 0.5in = 48px 
width:0.5in;
박스 0.5in

상대 단위 – 글꼴 크기 기반

단위설명코드데모
% 부모 요소 크기를 기준으로 비율 적용.
장점: 반응형에 적합
주의: 속성마다 기준이 다를 수 있음
예시 계산: 부모 너비 300px → 50% = 150px 
width:50%;
50%
부모 300px → 150px
em 현재 요소의 font-size 기준.
장점: 글꼴 크기에 비례
주의: 중첩 시 누적되어 예측 어려움
예시 계산: font-size:16px → 1em=16px, font-size:20px → 1em=20px 
font-size:2em;
2em (32px)
rem 루트(html)의 font-size 기준.
장점: 전체 앱에서 일관성 유지
예시 계산: html 기본 16px → 1rem=16px, 2rem=32px 
font-size:2rem;
2rem (32px)
ch 현재 폰트에서 ‘0’ 문자 폭 기준.
장점: 코드/모노스페이스 글꼴에서 문자 수 기반 폭 설정에 유용
주의: 가변폭 폰트에서는 실제 텍스트 폭과 다를 수 있음
예시 계산: Consolas 폰트에서 20ch = 문자 20개 폭 
width:20ch;
20ch 폭
ex 소문자 ‘x’ 높이 기준.
장점: 폰트의 상대적 높이를 기준으로 크기 조정 가능
주의: 폰트마다 ‘x’ 높이가 달라 실제 크기 차이가 발생
예시 계산: 특정 폰트에서 1ex ≈ 0.5em 
font-size:5ex;
텍스트 5ex

상대 단위 – 뷰포트 기반

단위설명코드데모
vw 뷰포트 너비의 1%.
장점: 화면 크기에 따라 자동 반응
주의: 모바일 브라우저 UI 크롬에 따라 값이 달라질 수 있음
예시 계산: 뷰포트 너비 1200px → 1vw = 12px 
width:30vw;
30vw 박스
vh 뷰포트 높이의 1%.
장점: 세로 크기 기준 반응형에 유용
예시 계산: 뷰포트 높이 800px → 1vh = 8px 
height:20vh;
20vh 박스
vmin 뷰포트 너비·높이 중 작은 값 기준.
장점: 정사각형 유지에 유용
예시 계산: 1200×800px → 1vmin = 8px 
font-size:5vmin;
텍스트 5vmin
vmax 뷰포트 너비·높이 중 큰 값 기준.
장점: 큰 화면 기준으로 크기 조정
예시 계산: 1200×800px → 1vmax = 12px 
font-size:5vmax;
텍스트 5vmax

상대 단위 – 레이아웃(Grid) 기반

단위설명코드데모
fr Grid에서 남은 공간을 비율로 분배.
장점: 레이아웃을 직관적으로 구성 가능
예시 계산: 1fr 2fr 3fr → 전체 6등분 중 1:2:3 비율 
grid-template-columns:1fr 2fr 3fr;
1fr
2fr
3fr

절대 vs 상대 단위 요약

구분특징예시 단위
절대 단위환경과 무관하게 고정된 크기px, pt, cm, mm, in
상대 단위부모, 루트, 뷰포트 등 기준에 따라 변함%, em, rem, vw, vh, vmin, vmax, fr, ch, ex

실무 팁

  • 글꼴 크기와 간격은 rem 기반으로 통일하면 유지보수에 유리합니다.
  • 레이아웃은 fr, vw/vh를 활용해 반응형을 구현하세요.
  • 인쇄용 문서에는 pt, cm, mm, in 단위를 사용하세요.
  • em은 부모 요소에 따라 누적되므로 복잡한 구조에서는 rem을 권장합니다.
  • ch와 ex는 폰트 특성에 따라 달라질 수 있으므로 주의하세요.
