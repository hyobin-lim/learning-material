자바스크립트 기초 올인원 학습 문서

이 문서는 완전 초보가 자바스크립트를 처음부터 끝까지 이해할 수 있도록 개념 설명, 예제 코드, 실행 가능한 데모를 나란히 제공합니다. 버튼을 눌러 데모를 실행하고, 바로 옆 코드로 비교해 보세요.

용어는 “일상어로 충분히 이해”할 수 있게 풀어 씁니다. 어려운 내용은 작게 한 걸음씩 — 직접 눌러보고 결과를 눈으로 확인하세요.

웹의 큰 틀과 자바스크립트의 역할

웹페이지는 보통 세 가지가 함께 움직입니다: HTML: 구조 CSS: 모양 JavaScript: 동작 HTML이 뼈대라면, CSS는 옷, 자바스크립트는 행동입니다.

데모: 버튼 클릭으로 문장 바꾸기

안녕! 나는 HTML에 적힌 기본 문장이야.
<!-- HTML: 문장과 버튼 -->
<div id="introText">안녕! 나는 HTML에 적힌 기본 문장이야.</div>
<button onclick="demo_intro_change()">문장 바꾸기</button>

<!-- JavaScript: 동작(문장 바꾸기/되돌리기) -->
function demo_intro_change() {
  const el = document.getElementById("introText");
  el.textContent = "반가워! 이제 자바스크립트가 문장을 바꿨어.";
}
function demo_intro_reset() {
  const el = document.getElementById("introText");
  el.textContent = "안녕! 나는 HTML에 적힌 기본 문장이야.";
}

변수와 상수, 자료형

변수는 “데이터를 담아두는 상자”입니다. let은 변경 가능한 상자, const는 바꿀 수 없는 상자(상수)입니다. 자료형은 담긴 내용의 종류로 숫자, 글자, 참/거짓, 객체, 배열 등이 있습니다.

데모: 변수/상수 만들고 출력하기

// 변수/상수 선언과 자료형
function demo_var_show() {
  let name = "효빈";         // 문자열(String)
  let age = 25;              // 숫자(Number)
  const country = "대한민국"; // 상수(Const)

  const isAdult = age >= 19; // 불리언(Boolean)
  const hobbies = ["코딩", "독서"]; // 배열(Array)
  const user = { name, age }; // 객체(Object)

  document.getElementById("varOut").textContent =
    `name=${name}, age=${age}, country=${country}, isAdult=${isAdult}, ` +
    `hobbies=${hobbies.join("/")}, user.name=${user.name}`;
}
function demo_var_reset() {
  document.getElementById("varOut").textContent = "";
}

연산자: 계산하고 비교하기

더하기, 빼기 같은 산술 연산자와, 같은지 다른지 비교하는 비교 연산자가 있습니다. 문자열 붙이기(결합)도 자주 씁니다.

데모: 산술/비교/문자열 결합

function demo_ops() {
  const a = 10, b = 3;
  const sum = a + b;    // 13
  const diff = a - b;   // 7
  const mul = a * b;    // 30
  const div = a / b;    // 3.333...
  const mod = a % b;    // 나머지: 1

  const eq = (a === 10);      // true
  const neq = (b !== 5);      // true
  const gt = (a > b);         // true
  const str = "안녕" + " " + "세계"; // "안녕 세계"

  document.getElementById("opsOut").textContent =
    `sum=${sum}, diff=${diff}, mul=${mul}, div=${div.toFixed(2)}, mod=${mod}, ` +
    `eq=${eq}, neq=${neq}, gt=${gt}, str=${str}`;
}

조건문과 반복문

조건문은 “상황에 따라 다르게” 실행하고, 반복문은 “여러 번” 실행합니다. if/else, switch, for, while가 대표적입니다.

데모: 조건/반복으로 메시지 만들기

function demo_flow() {
  const score = 82;
  let grade;
  if (score >= 90) grade = "A";
  else if (score >= 80) grade = "B";
  else grade = "C";

  // for 반복
  let nums = [];
  for (let i = 1; i <= 5; i++) {
    nums.push(i);
  }

  // while 반복
  let count = 3;
  let msg = "";
  while (count > 0) {
    msg += "카운트 " + count + " ";
    count--;
  }

  document.getElementById("flowOut").textContent =
    `score=${score}, grade=${grade}, nums=${nums.join(", ")}, msg=${msg}`;
}

함수: 일을 묶어두고 재사용하기

함수는 “어떤 일을 묶어둔 코드 덩어리”입니다. 입력(매개변수)을 받아 결과(반환값)를 돌려줍니다. 화살표 함수는 간단한 문법으로 함수를 표현하는 방식입니다.

데모: 환영 메시지 함수

// 일반 함수
function greet(name) {
  return `안녕하세요, ${name}님!`;
}
// 화살표 함수
const add = (x, y) => x + y;

function demo_func() {
  const msg = greet("효빈");
  const sum = add(12, 30);
  document.getElementById("funcOut").textContent =
    `greet=${msg}, add=${sum}`;
}

스코프, 호이스팅, 클로저

스코프는 “변수가 살아있는 범위”, 호이스팅은 “선언이 끌어올려지는 현상”, 클로저는 “함수가 만들어질 때의 환경을 기억하는 능력”입니다. 너무 어렵게 느끼지 않아도 괜찮아요. 데모로 감각을 익혀봅시다.

데모: 클로저로 카운터 만들기

function makeCounter() {
  let n = 0; // 이 변수는 makeCounter 안에 "캡쳐"되어 외부에 직접 안 보입니다.
  return function () { // 반환된 함수가 n을 기억합니다(클로저)
    n++;
    return n;
  };
}

const counter = makeCounter();

function demo_counter_inc() {
  const val = counter();
  document.getElementById("scopeOut").textContent = `count=${val}`;
}
function demo_counter_reset() {
  // 새 카운터로 교체(초기화 효과)
  window.counter = makeCounter();
  document.getElementById("scopeOut").textContent = "count=0";
}

DOM 제어: HTML을 자바스크립트로 건드리기

DOM은 “브라우저가 HTML을 객체로 표현한 구조”입니다. getElementById, querySelector로 요소를 찾고, textContent나 style로 내용을/모양을 바꿉니다.

데모: 목록에 항목 추가/삭제

  • 기본 항목 A
  • 기본 항목 B
function demo_dom_add() {
  const li = document.createElement("li"); // 새 요소 생성
  li.textContent = "새 항목 " + Math.floor(Math.random() * 100);
  document.getElementById("domList").appendChild(li); // 목록에 붙이기
}
function demo_dom_remove() {
  const list = document.getElementById("domList");
  if (list.lastElementChild) list.removeChild(list.lastElementChild);
}

이벤트: 사용자의 행동에 반응하기

클릭, 입력, 스크롤 같은 “사건”을 이벤트라고 부릅니다. 자바스크립트는 addEventListener로 이벤트에 반응하는 코드를 등록합니다.

데모: 입력하면 실시간 반영

const input = document.getElementById("evtInput");
const out = document.getElementById("evtOut");

input.addEventListener("input", () => {
  out.textContent = `입력: ${input.value}`;

배열: 데이터 모음과 반복 처리

배열은 데이터의 목록입니다. map, filter, reduce는 많이 쓰이는 “배열을 다루는 도구”입니다.

데모: 점수 변환/필터/합계

function demo_arr() {
  const scores = [72, 88, 91, 60, 45];

  const plus5 = scores.map(s => s + 5);          // 각 점수 +5
  const passed = plus5.filter(s => s >= 70);      // 70 이상 통과
  const sum = passed.reduce((acc, cur) => acc + cur, 0); // 합계

  document.getElementById("arrOut").textContent =
    `원본=${scores.join(", ")} | +5=${plus5.join(", ")} | 통과=${passed.join(", ")} | 합계=${sum}`;
}

객체: 관련 데이터와 기능 묶음

객체는 속성과 메서드를 가진 “한 덩어리”입니다. 구조 분해 할당으로 필요한 것만 꺼내 쓰기도 합니다.

데모: 객체 만들고 메서드 호출

function demo_obj() {
  const user = {
    name: "효빈",
    age: 25,
    greet() {
      return `안녕, 나는 ${this.name}이고 나이는 ${this.age}야.`;
    }
  };
  const { name, age } = user; // 구조 분해 할당
  document.getElementById("objOut").textContent =
    `${user.greet()} (구조분해: name=${name}, age=${age})`;
}

JSON과 로컬 저장소

JSON은 “데이터를 글자로 표현한 형식”입니다. 로컬 저장소(localStorage)에 브라우저에 데이터를 간단히 저장할 수 있습니다.

데모: 저장/불러오기

function demo_store_save() {
  const data = { name: "효빈", theme: "dark", lang: "ko" };
  localStorage.setItem("settings", JSON.stringify(data));
  document.getElementById("storeOut").textContent = "저장 완료";
}
function demo_store_load() {
  const raw = localStorage.getItem("settings");
  const data = raw ? JSON.parse(raw) : null;
  document.getElementById("storeOut").textContent =
    data ? `불러옴: ${data.name}/${data.theme}/${data.lang}` : "데이터 없음";
}
function demo_store_clear() {
  localStorage.removeItem("settings");
  document.getElementById("storeOut").textContent = "삭제 완료";
}

비동기 처리: fetch와 async/await

“잠시 시간이 걸리는 일”을 비동기로 처리합니다. fetch는 서버에 요청하고, async/await는 읽기 쉬운 비동기 문법입니다.

인터넷이 연결되지 않거나 CORS 제한이 있으면 실패할 수 있습니다. 실패도 학습 포인트!

데모: 공개 API에서 데이터 받아오기

async function demo_fetch() {
  const out = document.getElementById("fetchOut");
  out.textContent = "불러오는 중...";
  try {
    const res = await fetch("https://api.github.com/repos/javascript-tutorial/en.javascript.info/commits?per_page=3");
    if (!res.ok) throw new Error("요청 실패: " + res.status);
    const data = await res.json();
    const list = data.map(c => c.commit.message.split("\n")[0]).join(" | ");
    out.textContent = "최근 커밋 메시지: " + list;
  } catch (err) {
    out.textContent = "에러: " + err.message;
  }
}

에러 처리: try/catch

에러는 “예상 못한 문제”입니다. try/catch로 잡아내고 안전하게 메시지를 보여줄 수 있습니다.

데모: 의도적으로 에러 발생시키기

function demo_error() {
  const out = document.getElementById("errorOut");
  try {
    // 존재하지 않는 함수를 호출해서 에러 발생
    notDefinedFunction();
  } catch (e) {
    out.textContent = "잡은 에러: " + e.message;
  } finally {
    // 선택: 항상 실행되는 영역
    console.log("finally: 정리 작업 가능");
  }
}

클래스: 설계도 만들기

클래스는 “객체를 만드는 설계도”입니다. new로 찍어내고, 속성/메서드를 통해 동작을 정의합니다.

데모: 클래스 인스턴스 만들기

class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
  greet() {
    return `안녕, 나는 ${this.name}(${this.age})이야`;
  }
}
function demo_class() {
  const p = new Person("효빈", 25);
  document.getElementById("classOut").textContent = p.greet();
}

모듈: 파일을 나눠 관리하기

모듈은 “코드를 파일로 나눠서” 가져다 쓰는 방식입니다. 브라우저에선 <script type="module">로 사용합니다.

데모: 모듈로 함수 가져오기

아래 데모는 페이지 안에 모듈 스크립트를 “임베드”해서 보여주는 예입니다. 

<script type="module">
  // 내보내기(export)
  export function double(x) {
    return x * 2;
  }
</script>

<script type="module">
  import { double } from "./module.js"; // 실제론 별도 파일에서 가져옵니다.
  console.log(double(5)); // 10
</script>
실제 파일 분할 데모는 서버/번들러 환경이 있으면 더 자연스럽게 테스트됩니다.

디버깅: 버그 찾기와 개발자 도구

console.log로 상태를 출력하고, 브라우저 개발자 도구의 Sources 탭에서 중단점(breakpoint)을 걸어 흐름을 살펴볼 수 있습니다.

데모: 로그로 상태 확인

function demo_debug() {
  const state = { step: 1, ok: true };
  console.log("현재 상태:", state);
  document.getElementById("debugOut").textContent =
    "개발자 도구 콘솔을 열어 로그를 확인하세요 (F12)";
}

베스트 프랙티스(좋은 습관)

  • 명확한 이름: 변수/함수 이름을 “무엇을 하는지” 보이게 짓기
  • 작게 나누기: 긴 함수를 작은 기능으로 쪼개 재사용하기
  • 불변성 지키기: 가능하면 원본 배열/객체를 직접 수정하지 말고 복사해 쓰기
  • 에러 처리: 네트워크/파싱 같은 취약 지점에 try/catch 준비하기
  • 주석과 예제: 미래의 나(또는 동료)를 위해 의도를 적고 작게 데모 만들기
