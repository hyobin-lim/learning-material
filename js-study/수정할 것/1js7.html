자바(Java)와 자바스크립트(JavaScript) 비교

자바에서 백틱(`)

자바(Java)에서는 문자열을 큰따옴표(")로 감싸고, 문자 하나는 작은따옴표(')로 표현합니다.

String text = "Hello Java!";
char letter = 'A';

➡ 백틱(`)은 자바 문법에서 사용되지 않습니다.

자바스크립트에서 백틱(`)

자바스크립트(JavaScript)에서는 백틱(`)을 템플릿 리터럴(template literal)이라고 하며, 문자열을 더 편리하게 작성할 수 있습니다.

let name = "효빈";
let msg = `안녕하세요, ${name}님!`;
console.log(msg); // 출력: 안녕하세요, 효빈님!

➡ 백틱을 사용하면 ${변수} 형태로 문자열 안에 변수를 쉽게 삽입할 수 있습니다.

자바와 자바스크립트의 차이점

구분 자바(Java) 자바스크립트(JavaScript)
언어 성격 컴파일 언어 (JVM에서 실행) 인터프리터 언어 (브라우저/Node.js에서 실행)
용도 백엔드, 모바일 앱(Android), 대규모 시스템 웹 프론트엔드, 동적 UI, 서버(Node.js)
문자열 표현 큰따옴표("), 작은따옴표(') 큰따옴표("), 작은따옴표('), 백틱(`)
변수 선언 int x = 10;, String s = "hi"; let x = 10;, const s = "hi";
실행 환경 JVM(Java Virtual Machine) 브라우저, Node.js
주요 특징 정적 타입, 강력한 OOP 지원 동적 타입, 유연한 문법

정리

  • 자바에서는 백틱(`)을 사용하지 않는다.
  • 자바스크립트에서는 백틱(`)을 템플릿 리터럴로 사용한다.
  • 자바와 자바스크립트는 이름은 비슷하지만, 언어 성격과 용도가 크게 다르다.
