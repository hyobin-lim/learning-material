자바스크립트의 시작 — Chapter 01

이 문서는 자바스크립트의 역할, 브라우저 삽입 방법, 기본 용어, 브라우저가 코드를 해석하는 과정, 그리고 기초 문법 요소(변수, 데이터 타입, 연산자, 네이밍 규칙, 주석, 출력 방법, 스코프, typeof, 템플릿 리터럴, 기본 에러)를 설명합니다.

1. 자바스크립트의 역할

  • 웹의 요소 제어: HTML은 구조, CSS는 디자인, JS는 동작과 데이터 변경을 담당합니다.
  • 웹 애플리케이션 제작: 지도 길찾기, 로그인 인증, 웹 게임, 문서 편집 등 브라우저에서 애플리케이션처럼 동작합니다.
  • 다양한 라이브러리 사용: React, Vue, Angular, jQuery 등 풍부한 생태계가 있습니다.
  • 서버 개발: Node.js로 백엔드 개발까지 가능—프런트+백엔드를 모두 다루는 풀스택 개발자로 성장할 수 있습니다.

2. 브라우저에 삽입하기

브라우저는 자바스크립트 해석기(Interpreter)를 내장하고 있어, HTML 문서에 <script> 태그로 코드를 삽입하면 실행됩니다.

2.1 내부 스크립트 예제

<script>
    var heading = document.querySelector('#heading');
    heading.onclick = function() {
      heading.style.color = "red";
    };
</script>

2.2 외부 스크립트 예제

<script src="js/change-color.js"></script>

2.3 <script> 위치와 로딩 전략

  • 권장 위치: </body> 직전
  • defer: 문서 파싱 완료 후 실행
  • async: 다운로드 완료 즉시 실행

3. 자바스크립트의 기본 용어

  • 식(Expression): 값을 만들어낼 수 있는 모든 것
  • 문(Statement): 명령을 수행하는 구문

4. 기초 문법 요소

4.1 변수 선언

  • var: 오래된 방식, 함수 스코프
  • let: 블록 스코프, 값 재할당 가능
  • const: 블록 스코프, 값 재할당 불가

4.2 데이터 타입

  • 숫자(Number)
  • 문자열(String)
  • 불(Boolean)
  • null / undefined
  • 객체(Object)

4.3 연산자

  • 산술: + - * / %
  • 대입: = += -= *= /=
  • 비교: === !== < > <= >=
  • 논리: && || !
  • 삼항: 조건 ? 값1 : 값2

4.4 네이밍 규칙

  • 대소문자 구분 (case-sensitive)
  • 변수/함수: camelCase (예: addEventListener)
  • 클래스/생성자: PascalCase (예: EventListener)
  • 상수: 대문자 + 언더스코어 (예: MAX_VALUE)

4.5 주석(Comment)

// 한 줄 주석
/*
  여러 줄 주석
*/

4.6 출력 방법

  • alert("메시지"): 팝업 창
  • console.log("메시지"): 개발자 콘솔 출력
  • document.write("메시지"): HTML 문서에 직접 출력

4.7 스코프와 호이스팅

  • var: 함수 스코프, 선언이 끌어올려짐(호이스팅)
  • let/const: 블록 스코프, 호이스팅되지만 초기화 전 접근 불가

4.8 typeof 연산자

typeof 123;        // "number"
typeof "hello";    // "string"
typeof true;       // "boolean"
typeof undefined;  // "undefined"
typeof null;       // "object" (버그)

4.9 템플릿 리터럴

let name = "효빈";
console.log(`안녕하세요, ${name}님!`);

4.10 기본 에러

  • SyntaxError: 문법 오류
  • ReferenceError: 정의되지 않은 변수 참조
  • TypeError: 잘못된 타입 사용

5. 브라우저가 스크립트를 해석하는 과정

  1. HTML 파싱
  2. CSS 파싱
  3. JS 실행
  4. 렌더링

6. 자바스크립트의 장점과 한계

  • 장점: 모든 브라우저에서 실행 가능, 방대한 라이브러리 생태계, 프런트/백엔드 모두 활용 가능
  • 한계: 브라우저마다 동작 차이가 있을 수 있음, 보안에 취약할 수 있어 주의 필요

7. 실습 과제

아래 과제를 직접 시도해 보세요:

  • 버튼을 클릭하면 배경색이 바뀌도록 코드 작성하기
  • 사용자 이름을 입력받아 화면에 출력하기
  • 조건문을 사용해 입력한 숫자가 짝수인지 홀수인지 판별하기
  • typeof를 활용해 입력값의 타입을 확인해보기
  • 템플릿 리터럴을 사용해 "안녕하세요, ○○님!" 출력하기
