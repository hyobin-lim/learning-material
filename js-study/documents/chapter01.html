자바스크립트의 시작 — Chapter 01

이 문서는 자바스크립트의 역할, 브라우저 삽입 방법, 기본 용어, 브라우저가 코드를 해석하는 과정을 자세히 설명합니다. 각 예제에는 실행 결과 설명을 포함했습니다.

1. 자바스크립트의 역할

  • 웹의 요소 제어: HTML은 구조, CSS는 디자인, JS는 동작과 데이터 변경을 담당합니다.
  • 웹 애플리케이션 제작: 지도 길찾기, 로그인 인증, 웹 게임, 문서 편집 등 브라우저에서 애플리케이션처럼 동작합니다.
  • 다양한 라이브러리 사용: React, Vue, Angular, jQuery 등 풍부한 생태계가 있습니다.
  • 서버 개발: Node.js로 백엔드 개발까지 가능—프런트+백엔드를 모두 다루는 풀스택 개발자로 성장할 수 있습니다.
참고 링크: ECMAScript 표준 · 모던 자바스크립트 튜토리얼 · MDN JavaScript

2. 브라우저에 삽입하기

브라우저는 자바스크립트 해석기(Interpreter)를 내장하고 있어, HTML 문서에 <script> 태그로 코드를 삽입하면 실행됩니다.

2.1 내부 스크립트 예제

<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <title>글자색 바꾸기</title>
</head>
<body>
  <h1 id="heading">자바스크립트</h1>
  <p id="text">위 텍스트를 클릭해 보세요</p>

  <script>
    // document.querySelector('#heading') → id가 'heading'인 요소를 찾아 반환
    var heading = document.querySelector('#heading');

    // 요소에 클릭 이벤트 핸들러를 등록합니다.
    heading.onclick = function() {
      // 클릭 시 글자색을 빨간색으로 변경합니다.
      heading.style.color = "red";
    };
  </script>
</body>
</html>
실행 결과: 페이지가 열리면 "자바스크립트"라는 파란색 글자가 보입니다. 글자를 클릭하면 빨간색으로 바뀝니다.
Tip: 내부 스크립트는 빠르게 테스트하기 좋지만, 규모가 커지면 유지보수가 어려워집니다.

2.2 외부 스크립트 예제

<!-- HTML 문서 -->
<script src="js/change-color.js"></script>
// js/change-color.js (외부 파일)
// 외부 파일로 분리하면 재사용과 캐싱, 유지보수에 유리합니다.

document.addEventListener('DOMContentLoaded', function() {
  var heading = document.querySelector('#heading');
  if (!heading) return; // 요소가 없을 경우 방어 코드

  heading.addEventListener('click', function() {
    heading.style.color = 'red';
  });
});
실행 결과: 내부 예제와 동일하게 "자바스크립트" 글자를 클릭하면 빨간색으로 바뀝니다. 코드가 외부 파일에 분리되어 관리가 더 편리합니다.

2.3 <script> 위치와 로딩 전략

  • 권장 위치: </body> 직전—DOM 요소가 로드된 후 실행되어 오류를 줄입니다.
  • defer: HTML 파싱을 막지 않고, 문서 파싱 완료 후 순서대로 실행합니다.
  • async: 다운로드 완료 즉시 실행(순서 보장 없음)—분리 가능한 스크립트에 적합합니다.

3. 자바스크립트의 기본 용어

자바스크립트의 명령은 크게 식(Expression)과 문(Statement)으로 나뉩니다.

  • 식: 값을 만들어낼 수 있는 모든 것(변수에 저장 가능)
  • 문: 명령을 수행하는 구문(보통 끝에 ;), 조건문·반복문 등
// 아래는 모두 '식'입니다. 값으로 평가됩니다.
inch * 2.54;      // 연산식: inch 값을 cm로 변환하는 계산식
('안녕하세요');   // 문자열 리터럴: 문자열 자체가 값
5;                // 숫자 리터럴: 숫자 자체가 값

// '문'은 보통 제어 흐름이나 선언 등 명령을 수행합니다.
var x = 5;        // 변수 선언문(문) + 할당식(식)을 포함
if (x > 3) {      // 조건문(문)
  console.log('x는 3보다 큽니다'); // 함수 호출문(문)
}
실행 결과: 콘솔에 x는 3보다 큽니다가 출력됩니다.

4. 브라우저가 스크립트를 해석하는 과정

  1. HTML 파싱: <!DOCTYPE html> 확인 후, <html> 내부의 <head>, <body> 구조와 포함 관계를 파싱합니다.
  2. CSS 파싱: <style> 또는 외부 CSS를 파싱해 스타일 규칙을 적용합니다.
  3. JS 실행: <script>를 만나면 내장된 자바스크립트 해석기로 코드를 전달해 실행합니다.
  4. 렌더링: 파싱된 HTML과 CSS를 화면에 표시하고, 자바스크립트가 DOM을 조작하면 그 결과를 반영합니다.
실행 결과: HTML과 CSS가 먼저 화면에 표시된 후, 자바스크립트가 실행되어 요소의 동작(예: 클릭 시 색상 변경)이 반영됩니다.
Tip: DOM 조작이 많은 스크립트는 defer 속성이나 DOMContentLoaded 이벤트를 활용하면 안전합니다.

5. 자바스크립트의 장점과 한계

  • 장점: 모든 브라우저에서 실행 가능, 방대한 라이브러리 생태계, 프런트/백엔드 모두 활용 가능
  • 한계: 브라우저마다 동작 차이가 있을 수 있음, 보안에 취약할 수 있어 주의 필요

6. 실습 과제

아래 과제를 직접 시도해 보세요:

  • 버튼을 클릭하면 배경색이 바뀌도록 코드 작성하기
  • 사용자 이름을 입력받아 화면에 출력하기
  • 조건문을 사용해 입력한 숫자가 짝수인지 홀수인지 판별하기
