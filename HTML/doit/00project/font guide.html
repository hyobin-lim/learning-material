웹사이트 제작 시 폰트 사용 가이드

웹폰트와 시스템 기본 폰트를 상업용 웹사이트에서 안전하게 사용하는 방법을 표와 요약으로 정리했습니다.

웹폰트 vs 시스템 기본 폰트

구분 예시 폰트 상업용 사용 시 주의사항
웹폰트 (다운로드해서 쓰는 것) Noto Sans KR, Pretendard, SCoreDream, xeicon 무료 오픈소스/배포용 라이선스가 제공됨. 웹서버에 올려서 @font-face로 배포해도 안전. 단, 각 폰트의 라이선스 문서를 확인하는 것을 권장.
시스템 기본 폰트 (운영체제 내장) Arial, Verdana, Tahoma, 굴림, 돋움, AppleGothic, Dotum, Gulim, UnDotum, 본고딕, -apple-system, system-ui, sans-serif CSS에서 font-family로 지정하는 건 안전하며 상업용 웹사이트에서도 문제 없음. ⚠️ 단, 폰트 파일을 직접 추출해 서버에 올려 배포하면 라이선스 위반 가능성 있음. 반드시 사용자 기기에 설치된 폰트를 불러오는 방식만 사용.
핵심 요약
  • 웹폰트: 무료 배포용이므로 상업용 웹사이트에서 자유롭게 사용 및 배포 가능.
  • 시스템 기본 폰트: CSS로 지정해 사용자 기기에서 불러오는 건 안전. 폰트 파일을 직접 배포하면 라이선스 위반 위험.

즉, 웹사이트 제작 시에는 웹폰트는 자유롭게 배포 가능, 시스템 기본 폰트는 배포하지 말고 CSS로만 지정하면 됩니다.

참고: 각 폰트의 라이선스(예: SIL OFL, Apache-2.0 등)는 버전에 따라 달라질 수 있으므로, 사용 전 공식 저장소 또는 배포 페이지에서 라이선스 조항을 확인하세요.
