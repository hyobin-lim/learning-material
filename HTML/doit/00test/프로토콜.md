# HTTP와 FTP 프로토콜

## HTTP (HyperText Transfer Protocol)

**웹페이지를 주고받는 규칙**입니다. 여러분이 웹브라우저에서 사이트를 볼 때 사용됩니다.

- 요청하면 응답받는 방식
- 한번 주고받으면 연결 끊김
- 주로 웹페이지, 이미지, 데이터 전송
- 포트: 80 (HTTP), 443 (HTTPS)

**쉬운 비유**: 식당에서 주문하고 음식 받으면 끝나는 것처럼, 데이터 요청하고 받으면 연결이 끝납니다.

## FTP (File Transfer Protocol)

**파일을 업로드/다운로드하는 규칙**입니다. 서버에 파일을 올리거나 가져올 때 사용됩니다.

- 파일 전송 전용
- 연결 유지하며 여러 작업 가능
- 폴더 탐색, 파일 관리 가능
- 포트: 21

**쉬운 비유**: 도서관에 들어가서 책을 빌리고 반납하고 계속 이용하는 것처럼, 연결 상태를 유지하며 파일을 관리합니다.

## 클라이언트 프로그램

### HTTP 클라이언트
- Chrome, Firefox, Safari (웹브라우저)
- Postman (API 테스트)
- curl (명령줄 도구)
- wget (다운로드 도구)

### FTP 클라이언트
- FileZilla (가장 많이 사용, 무료)
- WinSCP (Windows 전용)
- Cyberduck (Mac/Windows)
- FTP 탐색기 (브라우저에서 ftp:// 주소 입력)

### 웹개발자 추천
- VS Code 확장: SFTP (FTP 통합)
- FileZilla (UI가 직관적)
