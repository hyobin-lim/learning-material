블록 레벨 요소와 인라인 레벨 요소 사용 가이드

HTML 요소는 크게 블록 레벨 요소와 인라인 레벨 요소로 나뉩니다. 각각의 특성과 사용 시점을 아래 표로 정리했습니다.

블록 레벨 요소

특징 사용 시점 예시 태그
항상 새로운 줄에서 시작하며 기본적으로 전체 너비를 차지 문서의 구조를 나눌 때, 큰 레이아웃이나 구획을 정의할 때 <div>, <p>, <h1>~<h6>, <section>, <article>
다른 블록 요소와 인라인 요소를 포함할 수 있음 제목, 문단, 목록, 표 등 독립적인 콘텐츠 단위를 표현할 때 <ul>, <ol>, <li>, <table>, <form>

인라인 레벨 요소

특징 사용 시점 예시 태그
줄 안에서 이어지며 필요한 만큼의 너비만 차지 텍스트 일부를 꾸미거나 강조할 때 <span>, <strong>, <em>, <a>
블록 요소 안에서 사용되며 다른 인라인 요소와 나란히 배치 가능 문장 중간에 링크, 이미지, 입력 필드 등을 삽입할 때 <img>, <br>, <input>, <label>

정리

  • 블록 레벨 요소: 문서의 큰 구조나 레이아웃을 정의할 때 사용합니다.
  • 인라인 레벨 요소: 문장이나 텍스트 흐름 안에서 특정 부분을 꾸미거나 기능을 추가할 때 사용합니다.
  • 즉, 큰 틀은 블록 요소로 잡고, 세부적인 꾸밈이나 기능은 인라인 요소로 처리하는 것이 일반적인 패턴입니다.

사용 시 주의할 점

  • 블록 레벨 요소는 다른 블록 요소와 인라인 요소를 포함할 수 있지만, 인라인 요소는 블록 요소를 포함할 수 없습니다.
  • 인라인 요소는 레이아웃을 나누는 용도로 사용하지 말고, 텍스트나 콘텐츠 일부를 꾸미는 용도로만 사용해야 합니다.
  • 블록 요소는 기본적으로 줄바꿈이 발생하므로, 불필요하게 중첩하거나 남용하지 않도록 주의합니다.
  • 인라인 요소는 줄 안에서 배치되므로, 너비와 높이를 직접 지정하는 데 제약이 있을 수 있습니다.
  • 시맨틱 태그(<header>, <footer>, <nav> 등)를 적절히 활용하면 구조와 의미를 명확히 할 수 있습니다.
  • CSS로 블록/인라인 성격을 변경할 수 있지만, 본래의 의미와 용도를 고려하여 사용하는 것이 바람직합니다.
